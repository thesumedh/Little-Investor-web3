LittleInvestors — Learn Blockchain in 7 Days
Vision
LittleInvestors is an interactive blockchain education platform built on Stellar that teaches users blockchain concepts through real on-chain experiences instead of passive reading or videos.
Most blockchain courses teach theory.
LittleInvestors teaches by doing.
Users connect a real Stellar wallet, complete lessons, pass quizzes, perform blockchain interactions, and earn a verifiable on-chain certificate upon course completion.
The goal is to make blockchain education practical, engaging, and accessible while driving real adoption of the Stellar ecosystem.

AI assessment
1. Connect Wallet Feature Check (mandatory)
The solution uses @stellar/stellar-sdk for wallet interactions. The backend relay handles transaction signing and ephemeral key generation. While Freighter is not directly used, the gasless design intentionally bypasses it; Stellar wallet connection is achieved via SDK-integrated key management.
2. Smart Contract Folder Structure Check (mandatory)
Found `contracts/vault/Cargo.toml` and `contracts/vault/src/lib.rs`. Missing `test.rs` and `Makefile` in the judged files, but the core Soroban contract structure is present and valid.
3. Smart Contract Code Validation (mandatory)
The contract is `AllowanceVault` with custom functions: initialize, set_limit, save, spend, reset_spent, get_details. It uses instance storage, custom data structures, and business logic for daily limits, savings, and spending. This is not the default Hello World.
4. README and Deployment Validation (mandatory)
README includes project title, description, architecture, and deployed contract details (Stellar Testnet, FeeBump sponsorship, transaction links to Stellar Expert). The contract ID is implied through the vault contract mentioned.
5. Smart Contract Integration Codebase Check (mandatory)
Integration is implemented in `src/controller/stellarRelayController.js` and `js/stellar-helper.js`; uses @stellar/stellar-sdk for RPC, transaction building, and FeeBump. The relay endpoint sponsors gas via FeeBump, matching the described architecture.
6. Cross-Check Contract and Frontend Function Matching (mandatory)
The contract functions (save, spend, get_details) are invoked by the frontend relay flow. The relayer builds transactions targeting these contract methods. Real contract interaction is confirmed via on-chain transaction logs.
